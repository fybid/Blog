数据库 问题收集

· 2018-07-09 · # 数据库

MySQL

  1. Data truncated for column XXXXX

    错误说明：程序中字段数据的长度 与 数据库字段 设定不一致

  2. too many connections

    错误说明：每次进行数据库操作都会开启连接，当连接数超过配置的数量限制就会报错

  3. Deadlock found when trying to get lock; try restarting transaction

    错误说明：死锁相关，处理方式已丢失..

  4. SQL Error: 1064, SQLState: 42000

    错误说明：数据库字段使用了to、from等保留关键字

SQLServer

  1. sqlserver:'@P0' 附近有语法错误

     错误说明：参数不符合关键字或方法传参的要求，P0表示第一个，P1表示第二个，以此类推。

  2. 对于特殊字符的处理

     解决方法：<![CDATA[此处进行特殊转义字符处理]]>，似乎mysql通用。
